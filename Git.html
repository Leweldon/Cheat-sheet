  • What is Git you may ask. Its a Version Control System for tracking files in your computer.
  • Repository is basically the meat of the whole project that you would be working on.
  • A Branch is a extension from the Repository.All Repositories start with a master branch.
  • Commit will be the main way you will change things in your Repository
  • Git commands to know for changing directory-Cd- (Change directory) Changes the directory or folder
  • Cd~-Will you from whereever you are in your GitBash to the directory.
  • CD..- Moves you up one directory
  • Ls- List all the files that it can find on the computer
  • Git commnads to learn for local Repositories and clone files
  • Git init- Creats new local Repositories in current directory
  • Git clone- makes a copy of a Repository from the main resource and also creatys a directory inside the current one to put the project in there.
  • Three Git commands that i think are most important to remember.
  • Git status- This will help you see what is happeneing overall with youe whole project. Tell you what has been added or removed and is truly helpful if you think you may have made a mistake
  • Git push- Takes the locally made changes that you made to a given branch and applies them to the local version
  • Git Branch . - Stages all chanages from the directory( lets you see the history of this directory)
